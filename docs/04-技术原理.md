## 一、整体架构

### 1.1 四层架构

```
deploy.py (CLI 入口，typer)
    ↓
glue/         — 编排层：串联 services，零业务逻辑
    ↓
services/     — 业务层：功能单元，调用 libs
    ↓
libs/         — 适配层：封装第三方库差异（飞书 SDK / OSS SDK / LLM SDK）
    ↓
config/       — 配置 + 数据模型（Pydantic）
```

依赖方向：**单向向下**，严格遵守。

- glue → services → libs → config
- service 不调另一个 service（同级不互通）
- service 不直接 import third_party SDK（必须走 libs）
- lib 单文件 50-100 行，封装一个关注点

### 1.2 三个真相源

| 真相源 | 存什么 | 谁写 | 谁读 |
|---|---|---|---|
| 飞书 bitable 三张表 | 课程元数据 + 资料记录 + 心得记录 | 学生表单 + 管理员 UI + sync | sync / docs / link |
| 阿里云 OSS | 源文件 + OCR 全文 + 摘要 md（重数据） | archive / ocr | docs / link |
| 飞书 drive | 摘要 md（轻数据，供问答） + 飞书原生 docx | ocr / docs | 用户访问 |

**本地无持久化**：bitable 是唯一运行时真相源，本地 json 不再存在。

## 二、数据流详解

### 2.1 完整链路（一份资料从提交到出现在文档）

```
[学生表单]                 [管理员 UI]
   │                          │
   ▼                          ▼
资料管理表（bitable）
   │
   │ 1. archive-materials
   │    feishu.download_file(file_token) → bytes
   │    cloud.upload(bytes, oss_key) → OSS
   │    cloud.download_url(oss_key) → 纯直链 URL
   │    feishu.update_bitable_record(record, {文件链接: url, 归档时间: ts})
   │    --purge-immediately → feishu.delete_file + 清空附件字段
   ▼
资料管理表（文件链接字段有值，已归档）
   │
   │ 2. ocr-materials
   │    从 OSS 下载源文件
   │    检测格式：
   │      PDF → 直接 OCR
   │      Word/Excel/PPT → LibreOffice 转 PDF → OCR
   │      zip → 解压 → 递归
   │    GLM-OCR 提取全文
   │    DeepSeek 摘要
   │    摘要 md → 飞书 drive + OSS
   │    回填资料表「资料摘要」字段
   ▼
资料管理表（摘要字段有值，已 OCR）
   │
   │ 3. sync
   │    扫三张表
   │    双层门控：资料表的课程不在主数据表 → 跳过 + 警告
   │    更新主数据表的「资料数量」+「最后更新」
   ▼
课程主数据表（资料数量 + 最后更新字段已更新）
   │
   │ 4. docs
   │    CourseDataService.get_all() 实时聚合：
   │      - 主数据表（基本字段）
   │      - 资料管理表（已通过审核 → materials[]）
   │      - 心得管理表（已通过审核 → insights[]）
   │      - 派生 contributors[]
   │    LLM 输入：materials[].summary + materials[].recommendation_reason + course.insights
   │    LLM 输出：overview（课程概述）+ material_synthesis（资料串讲）
   │    生成 docx + 写 5 类 block
   ▼
课程文档（飞书 docx，含可点击下载链接）
   │
   │ 5. link
   │    从 deploy_state 读 course_to_doc_map
   │    GET 学年文档顶层块 → 定位 nav_table（block_type=31）
   │    DELETE 旧 nav_table → POST 新 nav_table（含 doc_url）
   ▼
学年文档的导航表「学习指南」字段链接更新
```

### 2.2 三级存储链路（OSS + 飞书 drive）

| 层级 | 内容 | 用途 | 生命周期 |
|---|---|---|---|
| OSS raw/ | 源文件原件 | 长期归档，可恢复 | 永久（生产化清单 3.3 考虑生命周期） |
| OSS ocr/ | OCR 全文 md | 重数据归档 | 永久 |
| OSS summary/ | LLM 摘要 md | 重数据归档 + 供未来知识助手 | 永久 |
| 飞书 drive | 摘要 md（一份一份） | 供飞书内问答 / 浏览 | 与 OSS summary 同步 |
| 飞书 drive | 资料目录 catalog.md | 聚合所有摘要的目录 | 每次 catalog 命令重建 |
| 飞书 bitable | 资料表「资料摘要」字段 | 文本直接嵌入 bitable | 与 OSS summary 同步 |

## 三、关键代码路径

### 3.1 CLI → glue → service 的调用链

```
deploy.py（typer CLI）
    ↓ asyncio.run
glue/deploy.py:_deploy_xxx(settings, args)
    ↓
glue/pipeline.py:pipeline.xxx_pipeline()
    ↓ lazy import + 局部构造 adapter/service
services/xxx_service.py: XxxService.method()
    ↓
libs/feishu/xxx.py 或 libs/cloud/xxx.py（adapter）
    ↓
third_party SDK（lark-oapi / oss2 / httpx）
```

每个 pipeline 内部局部构造 FeishuAdapter，在 finally 关闭。Pipeline.__init__ 只持有 settings + rollback_manager。

### 3.2 双层门控

**门控位置 1（sync 层）**：`services/sync_service.py:sync()`
```python
for r in material_records:
    f = r.get("fields") or {}
    if _select_text(f.get("审核状态")) != "已通过":
        continue
    cn = _extract_course_name(f.get("课程"))
    if cn not in course_meta:
        # 门控：跳过 + 警告
        skipped_materials += 1
        warnings.append(f"课程 {cn} 不在主数据表...")
        continue
    materials_by_course[cn].append(f)
```

**门控位置 2（docs 层）**：`services/course_data_service.py:get_all()`
```python
for r in material_records:
    f = r.get("fields") or {}
    if _select_text(f.get("审核状态")) != "已通过":
        continue
    cn = _extract_course_name(f.get("课程"))
    if cn not in courses:   # courses 来自主数据表
        continue
    # ... 聚合到 materials_by_course[cn]
```

两层独立实现，互不依赖。即使直接跑 docs（跳过 sync），门控也生效。

### 3.3 CourseData 是运行时聚合

```python
# services/course_data_service.py
async def get_all(self, app_token: str) -> List[CourseData]:
    # 1. 主数据表 → CourseData 基本字段
    courses = {name: CourseData(...) for r in course_records}
    # 2. 资料管理表 → materials[]
    for r in material_records:
        if cn in courses and 审核通过:
            materials_by_course[cn].append(Material(...))
    # 3. 心得管理表 → insights[]
    for r in insight_records:
        if cn in courses and 审核通过:
            insights_by_course[cn].append(Insight(...))
    # 4. 派生 contributors[]
    for name, course in courses.items():
        course.materials = materials_by_course.get(name, [])
        course.insights = insights_by_course.get(name, [])
        course.contributors = self._aggregate_contributors(course.materials)
    return list(courses.values())
```

CourseData 不写本地文件。每次 docs / link / wiki 调用都重新聚合。

## 四、配置详解（.env 字段）

### 4.1 飞书

| 字段 | 说明 |
|---|---|
| `FEISHU_APP_ID` | 应用 ID（开放平台） |
| `FEISHU_APP_SECRET` | 应用 Secret |
| `FEISHU_BASE_URL` | API 入口，默认 `https://open.feishu.cn/open-apis` |
| `FEISHU_DOC_HOST` | 文档域名，私有部署填 `xyz.feishu.cn` |
| `WIKI_SPACE_NAME` | 知识空间名（首次创建用） |
| `BITABLE_APP_TOKEN` | 管理 bitable 的 app_token（init-bitable 后填回） |

### 4.2 LLM / OCR

| 字段 | 说明 |
|---|---|
| `LLM_API_KEY` | DeepSeek API Key |
| `LLM_BASE_URL` | OpenAI 兼容入口，默认 `https://api.deepseek.com` |
| `LLM_MODEL` | 模型名，默认 `deepseek-v4-pro` |
| `GLM_API_KEY` | 智谱 API Key（GLM-OCR） |
| `GLM_OCR_URL` | layout_parsing 接口 URL |

### 4.3 阿里云 OSS

| 字段 | 说明 |
|---|---|
| `CLOUD_DRIVE_BACKEND` | `aliyun_oss` 或 `local_stub` |
| `OSS_BUCKET` | bucket 名 |
| `OSS_ENDPOINT` | 入口，如 `https://oss-cn-beijing.aliyuncs.com` |
| `OSS_ACCESS_KEY_ID` / `OSS_ACCESS_KEY_SECRET` | AK/SK |
| `OSS_PRESIGNED_TTL` | 私有 sign_url 有效期（默认 86400s） |
| `OSS_AUTH_MODE` | `access_key`（个人）或 `ram_role`（企业 STS） |
| `OSS_ROLE_ARN` | ram_role 模式必填 |
| `OSS_CDN_DOMAIN` | CDN 域名（可选） |
| `OSS_PUBLIC_BASE` | **公共读直链 base**（bucket 改公共读后启用），如 `https://your-bucket.oss-cn-beijing.aliyuncs.com` |

`download_url` 优先级：`OSS_PUBLIC_BASE` > `OSS_CDN_DOMAIN` > 私有 sign_url

### 4.4 路径

| 字段 | 说明 |
|---|---|
| `MATERIALS_BASE` | 本地课程资料根目录（测试 / 历史资料） |
| `SUMMARY_DIR` | OCR 摘要本地缓存 |
| `CLOUD_STUB_DIR` | LocalStubDrive 本地占位目录 |
| `COURSE_REFORM_NOTES_DIR` | 课程教改笔记目录 |

### 4.5 日志

| 字段 | 说明 |
|---|---|
| `LOG_LEVEL` | `DEBUG` / `INFO` / `WARNING` / `ERROR` |
| `LOG_TO_FILE` | 是否写文件 |
| `LOG_FILE_PATH` | 文件路径 |

## 五、数据模型

### 5.1 bitable 三张表字段

**课程主数据表**：
- 课程名称（文本）
- 授课老师（文本）
- 开课学期（单选：大一上 / 大一下 / 大二上 / ...）
- 课程类型（单选：专业必修课 / 非专业必修课）
- 考试形式（单选：闭卷 / 开卷 / 论文 / 其他）
- 资料数量（数字，sync 派生）
- 最后更新（日期，sync 派生）

**资料管理表**：
- 贡献者 / 届别（学生填）
- 课程（单选，下拉从主数据表读）
- 资料类型（单选）
- 推荐理由（长文本）
- 文件附件（飞书附件字段 type 17）
- 资料名称 / 原始文件名 / 文件链接（系统回填）
- 资料摘要（系统回填，OCR 后 LLM 生成）
- 归档时间 / 上传时间（系统填）
- 审核状态（单选：待审核 / 已通过 / 已拒绝）

**心得管理表**：
- 作者 / 届别 / 课程 / 成绩 / 心得内容（学生填）
- 提交时间（系统填）
- 审核状态 / 审核人（管理员填）

详见 `config/course_schema.py`。

### 5.2 CourseData（运行时聚合对象）

```python
class CourseData(BaseModel):
    name: str
    teacher: str = ""
    semester: str = ""       # 大一上 / 大一下 / ...

    exam: str = ""
    insights: List[Insight]
    materials: List[Material]
    contributors: List[Contributor]
    doc_url: str = ""
    updated_at: str = ""
```

详见 `config/course_schema.py`。

## 六、二次开发

### 6.1 加新 CLI 命令

1. `services/` 加新 service 方法（或扩展现有）
2. `glue/pipeline.py` 加 `xxx_pipeline` 方法
3. `glue/deploy.py` 加 `_deploy_xxx` 函数 + 注册 `_MODE_HANDLERS`
4. `deploy.py` 加 `@app.command` typer 入口

### 6.2 改 prompt 模板

`config/prompts/` 下是 Jinja2 模板：
- `course_guide_v3.j2`：课程文档（overview + material_synthesis）
- `year_overview.j2`：学年总论

改模板后无需改代码，下次 docs 调用自动生效。

### 6.3 改数据模型

`config/course_schema.py` 改 Pydantic 模型。bitable 字段定义也在这个文件（`COURSE_TABLE_FIELDS` / `MATERIALS_TABLE_FIELDS` 等），改完后跑 `python deploy.py fix-bitable` 补字段。

### 6.4 加云盘后端

`libs/cloud/` 下加新 adapter（实现 `CloudDriveAdapter` 抽象类），`libs/cloud/__init__.py:get_drive` 加分支。

## 七、性能 / 成本

### 7.1 飞书 API 配额

- 免费版：10000 次/月
- 一次完整 sync（30 课 / 50 资料）：约 50 次 API
- 一次 archive（10 文件）：约 40 次 API
- 一次 ocr（10 文件）：约 10 次 API

按月 5 次完整 sync + 5 次 archive + 5 次 ocr 估算：~ 500 次/月，远低于配额。

### 7.2 LLM 成本

- DeepSeek：每 100 万 token 输入约 1 元，输出约 2 元
- 单次课程文档生成：约 2000-5000 token 输入 + 1000-2000 token 输出 = 约 0.01 元
- 单次摘要：约 5000-20000 token 输入 + 500 token 输出 = 约 0.02 元

按 100 课 + 200 资料估算：~ 5 元，可控。

### 7.3 OCR 成本

- GLM-OCR：每页约 0.01-0.03 元
- 单份 50 页 PDF：约 0.5-1.5 元

按 200 份 PDF 估算：~ 200 元，需要预算。

### 7.4 OSS 存储

- 标准 storage：0.12 元/GB/月
- 流量：内网免费，外网 0.5 元/GB
- 100 GB 资料 + 月 10 GB 流量：约 17 元/月
