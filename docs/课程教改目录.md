## 一、项目概述

### 1.1 真正的背景与问题

**传统大礼包的痛点**：
- PPE专业每届学习委员手工整理"大礼包"（课程资料压缩包）
- 手工整理繁琐枯燥，学委整理意愿不强
- 一届一届传下来，内容陈旧、结构杂乱
- 不好用、不透明，学弟学妹不知道从何开始

**现有样本**：
- 9门课程、127个文件、471.72 MB
- 结构不一致（有的按年份分，有的按资料类型分）
- 缺少元信息（谁贡献的、推荐理由、学习心得）
- 版本混乱（同一门课有多年版本，不知道用哪个）

### 1.2 项目愿景

构建一个**云端智能大礼包**：
- 学长学姐可以**自己上传资料**（结构化填写：身份、推荐理由、资料原件）
- **AI Agent主动整理**，分课程归档，做好重命名（如"课程笔记_23级小陈"）
- 基于**高分学长学姐的心得体会**，将资料"串讲"成系统、流畅、可读的飞书文档
- 每个参与者有**强烈的参与感和满足感**

### 1.3 核心价值

**贡献者列表不仅记录"上传了什么"，还记录"如何丰富了我们对课程的理解"**

例如：
> | 贡献者 | 贡献 |
> |--------|------|
> | 22级小王 | 提倡在计量经济学课程中，PPE的同学应该系统自学python，尤其是pandas包，以培养数据分析的直觉，并上传了优秀的简明教材。|

### 1.4 核心资产

**高分学长学姐的心得体会**——它们的质量和内容结构直接决定课程文档的质量。

各级班委（尤其是掌握班级成绩数据的学委）会判断和邀请撰写心得体会。

### 1.5 目标用户

| 属性 | 值 |
|------|-----|
| 人数 | 约27人 |
| 阶段 | 大一学期开始 |
| 专业 | PPE实验班 |
| 场景 | 课前预习、课后复习、期末备考 |
| 角色 | 学长学姐（贡献者）、学弟学妹（使用者）、宣传委员（审批者） |

## 二、核心功能（修订）

### 2.1 功能优先级重排

| 优先级 | 功能 | 说明 |
|--------|------|------|
| 🔴 P0 | 结构化上传 | 学长学姐上传资料+填写身份+推荐理由 |
| 🔴 P0 | AI整理归档 | 分课程归档，重命名（资料名_贡献者） |
| 🔴 P0 | 课程文档生成 | 基于心得体会生成6部分课程文档 |
| 🔴 P0 | 贡献者列表 | 自动生成贡献者表格 |
| 🟡 P1 | 审批流程 | 宣传委员审批修订意见 |
| 🟡 P1 | 操作日志 | 所有提议、采纳、编辑留痕 |
| 🟢 P2 | AI问答 | 基于课程资料的智能问答（辅助功能） |
| 🟢 P2 | 班级群聊 | 飞书群内的AI助手（辅助功能） |

### 2.2 课程文档结构（固定模板）

每门课程的飞书文档包含以下6个部分，**全部基于高分学长学姐的心得体会生成**：

1. **课程内容概述**
2. **学习难点**
3. **老师偏好**（藏在考核要求之下的偏好，决定努力方向）
4. **资料分类列表**（方便已明确需求的学弟学妹按类别快速找到）
5. **资料串讲**（基于心得体会，将资料有机串联）
6. **贡献者列表**（表格形式：贡献者 | 贡献描述）

### 2.3 用户使用体验

> 打开一个名为"大二上课程"的飞书docs，看到一个grid（类似notion的inline_database），列着"中国经济概论""西方政治思想史"等课程行，将鼠标放在第一列第一行上的那个格子，点open，就来到"中国经济概论"课程的专属页面，页面里基于高分学生的心得体会串着各种学习资料。

```python
# services/upload_service.py

class UploadService:
    """资料收集服务"""
    
    async def process_upload(
        self,
        uploader_id: str,           # 上传者ID（如"23级小陈"）
        uploader_grade: str,        # 年级
        course_name: str,           # 课程名称
        material_type: str,         # 资料类型（PPT/笔记/真题/阅读材料等）
        recommendation_reason: str, # 推荐理由
        file_content: bytes,        # 文件内容
        file_name: str              # 原始文件名
    ) -> dict:
        """
        处理资料上传
        
        流程：
        1. 重命名文件：{资料类型}_{上传者}_{原始文件名}
        2. 上传到飞书云空间
        3. 记录元信息到多维表格
        4. 触发文档更新
        """
        
        # 1. 生成新文件名
        new_filename = self._generate_filename(
            material_type, uploader_id, file_name
        )
        
        # 2. 上传到飞书
        file_token = await self.feishu.upload_file(
            file_content, new_filename, course_name
        )
        
        # 3. 记录到多维表格
        record = await self.feishu.add_bitable_record(
            app_token=self.bitable_token,
            table_id=self.materials_table_id,
            fields={
                "资料名称": new_filename,
                "贡献者": uploader_id,
                "年级": uploader_grade,
                "课程": course_name,
                "资料类型": material_type,
                "推荐理由": recommendation_reason,
                "文件链接": f"feishu://file/{file_token}",
                "上传时间": datetime.now().isoformat()
            }
        )
        
        # 4. 触发课程文档更新
        await self.doc_generator.update_course_doc(course_name)
        
        return {
            "success": True,
            "file_token": file_token,
            "record_id": record["record_id"]
        }
    
    def _generate_filename(
        self, 
        material_type: str, 
        uploader_id: str, 
        original_name: str
    ) -> str:
        """生成标准化文件名"""
        # 例如："课程笔记_23级小陈_计量经济学重点.md"
        return f"{material_type}_{uploader_id}_{original_name}"
```

```python
# services/experience_service.py

class ExperienceService:
    """心得体会管理服务"""
    
    async def submit_experience(
        self,
        author_id: str,             # 作者ID（如"22级小王"）
        author_grade: str,          # 年级
        course_name: str,           # 课程名称
        score: str,                 # 该课程成绩（用于验证"高分"）
        experience_content: str     # 心得体会内容
    ) -> dict:
        """
        提交心得体会
        
        心得体会是核心资产，需要：
        1. 结构化存储
        2. 提取关键信息（学习难点、老师偏好等）
        3. 用于生成课程文档
        """
        
        # 1. 存储心得体会
        record = await self.feishu.add_bitable_record(
            app_token=self.bitable_token,
            table_id=self.experiences_table_id,
            fields={
                "作者": author_id,
                "年级": author_grade,
                "课程": course_name,
                "成绩": score,
                "心得内容": experience_content,
                "提交时间": datetime.now().isoformat(),
                "状态": "待审核"  # 需要学委验证成绩真实性
            }
        )
        
        # 2. AI提取关键信息
        extracted_info = await self.llm.extract_course_info(
            experience_content, course_name
        )
        
        # 3. 更新课程知识库
        await self.knowledge_base.update_course_info(
            course_name, extracted_info
        )
        
        return {
            "success": True,
            "record_id": record["record_id"],
            "extracted_info": extracted_info
        }
    
    async def get_experiences_by_course(self, course_name: str) -> list:
        """获取某课程的所有心得体会（已审核通过的）"""
        records = await self.feishu.query_bitable(
            app_token=self.bitable_token,
            table_id=self.experiences_table_id,
            filter={
                "conditions": [
                    {"field": "课程", "operator": "is", "value": course_name},
                    {"field": "状态", "operator": "is", "value": "已审核"}
                ]
            }
        )
        return records
```

```python
# services/doc_generator.py

class CourseDocGenerator:
    """课程文档生成服务"""
    
    # 课程文档模板（6个部分）
    DOC_TEMPLATE = """
# {course_name}

## 一、课程内容概述
{course_overview}

## 二、学习难点
{learning_difficulties}

## 三、老师偏好
{teacher_preferences}

## 四、资料分类列表
{material_list}

## 五、资料串讲
{material_guide}

## 六、贡献者列表
{contributor_table}
"""
    
    async def generate_course_doc(self, course_name: str) -> str:
        """
        生成课程文档
        
        基于：
        1. 高分学长学姐的心得体会
        2. 上传的资料列表
        3. 贡献者信息
        """
        
        # 1. 获取该课程的所有心得体会
        experiences = await self.experience_service.get_experiences_by_course(
            course_name
        )
        
        # 2. 获取该课程的所有资料
        materials = await self.material_service.get_materials_by_course(
            course_name
        )
        
        # 3. AI生成6个部分的内容
        doc_content = await self._generate_sections(
            course_name, experiences, materials
        )
        
        # 4. 写入飞书文档
        doc_token = await self.feishu.create_or_update_doc(
            parent_node_token=self.get_course_node(course_name),
            title=course_name,
            content=doc_content
        )
        
        return doc_token
    
    async def _generate_sections(
        self, 
        course_name: str,
        experiences: list,
        materials: list
    ) -> str:
        """生成文档的6个部分"""
        
        # 合并所有心得体会
        combined_experience = "\n\n---\n\n".join([
            f"【{exp['作者']}】\n{exp['心得内容']}"
            for exp in experiences
        ])
        
        # AI生成各部分
        prompt = f"""
你是一个教学经验丰富的助教，需要基于学长学姐的心得体会，为"{course_name}"课程编写一份学习指南。

学长学姐的心得体会：
{combined_experience}

请生成以下6个部分的内容：

1. 课程内容概述（200-300字）
2. 学习难点（列出3-5个难点，每个难点简要说明）
3. 老师偏好（隐藏在考核要求下的偏好，决定努力方向）
4. 资料分类列表（根据上传的资料类型分类）
5. 资料串讲（基于心得体会，说明如何使用这些资料进行学习）
6. 贡献者列表（表格形式）

要求：
- 内容要基于心得体会，不要编造
- 语言流畅、可读性高
- 实用性强，能帮助学弟学妹
"""
        
        response = await self.llm.generate(prompt)
        
        # 格式化贡献者表格
        contributor_table = self._format_contributor_table(materials)
        
        # 组装完整文档
        return self.DOC_TEMPLATE.format(
            course_name=course_name,
            **response,
            contributor_table=contributor_table
        )
    
    def _format_contributor_table(self, materials: list) -> str:
        """格式化贡献者表格"""
        # 按贡献者分组
        contributors = {}
        for m in materials:
            name = m["贡献者"]
            if name not in contributors:
                contributors[name] = []
            contributors[name].append(m)
        
        # 生成表格
        rows = ["| 贡献者 | 贡献 |", "|--------|------|"]
        for name, mats in contributors.items():
            contribution = "、".join([
                f"上传{m['资料类型']}（{m['推荐理由'][:30]}...）"
                for m in mats[:3]  # 最多显示3个
            ])
            rows.append(f"| {name} | {contribution} |")
        
        return "\n".join(rows)
```

```python
# services/approval_service.py

class ApprovalService:
    """审批流程服务"""
    
    async def submit_revision_proposal(
        self,
        proposer_id: str,           # 提议者ID
        course_name: str,           # 课程名称
        revision_type: str,         # 修订类型（内容修订/资料增删/结构优化）
        revision_content: str,      # 修订内容
        reason: str                 # 修订理由
    ) -> dict:
        """
        提交修订提议
        
        所有修订意见需要宣传委员审批
        """
        
        # 1. 创建审批记录
        record = await self.feishu.add_bitable_record(
            app_token=self.bitable_token,
            table_id=self.revisions_table_id,
            fields={
                "提议者": proposer_id,
                "课程": course_name,
                "修订类型": revision_type,
                "修订内容": revision_content,
                "修订理由": reason,
                "状态": "待审批",
                "提交时间": datetime.now().isoformat()
            }
        )
        
        # 2. 通知宣传委员
        await self.notify_committee_member(
            committee_type="宣传委员",
            message=f"收到新的修订提议：{course_name} - {revision_type}"
        )
        
        return {"success": True, "record_id": record["record_id"]}
    
    async def approve_revision(
        self,
        revision_id: str,
        approver_id: str,
        approved: bool,
        comment: str = ""
    ) -> dict:
        """
        宣传委员审批修订
        """
        
        # 1. 更新审批状态
        await self.feishu.update_bitable_record(
            app_token=self.bitable_token,
            table_id=self.revisions_table_id,
            record_id=revision_id,
            fields={
                "状态": "已批准" if approved else "已拒绝",
                "审批人": approver_id,
                "审批意见": comment,
                "审批时间": datetime.now().isoformat()
            }
        )
        
        # 2. 如果批准，执行修订
        if approved:
            revision = await self.get_revision(revision_id)
            await self.doc_generator.apply_revision(revision)
        
        # 3. 记录操作日志
        await self.log_operation(
            operation_type="revision_approval",
            operator=approver_id,
            details={
                "revision_id": revision_id,
                "approved": approved,
                "comment": comment
            }
        )
        
        return {"success": True}
```
