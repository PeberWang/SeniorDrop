# 00 · 快速开始

> 目标读者：第一次部署的管理员，不要求懂代码。
>
> 跑完本文档后，你会有一个能用的飞书知识库骨架（4 个学年文档 + 课程导航 + 占位课程文档），但没有真实资料。真实资料通过表单收集 → 详见 [01-管理员指南.md](01-管理员指南.md)。
>
> **总耗时**：约 60-90 分钟（注册账号 + 装环境 + 配置 + 部署）

## 一、准备账号 + 装环境

按顺序做完 6 件事。每件事都是独立的，不会互相阻塞。

### 1.1 飞书企业版（用作知识库 + 多维表格载体）

**注册**：
1. 访问 https://www.feishu.cn/
2. 点「免费使用」→ 用手机号注册一个企业版（个人版也行，但企业版功能更全）
3. 完成企业认证（个人项目可填「XX 专业筹备组」等，不强求真实企业名）

**创建应用**：
1. 访问 https://open.feishu.cn/，用刚注册的飞书账号登录
2. 点「开发者后台」→「创建企业自建应用」
3. 填应用名（如「XX 专业课程大礼包」）、描述、图标（随便传一张）
4. 创建完成后，进入应用详情页 → 「凭证与基础信息」→ 看到 **App ID**（cli_xxx 开头）和 **App Secret**，复制保存

**开通权限**（应用详情页 → 「权限管理」→「权限管理」）：
搜索并开通以下 5 个权限：
- `wiki:wiki`（知识库读写）
- `docx:document`（文档读写）
- `bitable:app`（多维表格读写）
- `drive:drive`（云盘读写）
- `drive:media:upload`（媒体上传）

**发布应用**：应用详情页 → 「版本管理与发布」→ 创建版本 → 申请线上版本 → 通过（你是管理员，自己审核自己）。

### 1.2 阿里云 OSS（用作资料归档存储）

**注册 + 开通**：
1. 访问 https://www.aliyun.com/
2. 右上角「登录/注册」，用支付宝扫码或手机号注册
3. 完成实名认证（个人认证即可，免费）

**开通对象存储 OSS**：
1. 登录后，顶部搜索框输入「OSS」→ 点「对象存储 OSS」
2. 如未开通，会提示「立即开通」，点开通（免费开通，按使用付费）

**新建 Bucket**：
1. OSS 控制台左侧 → 「Bucket 列表」→ 点「创建 Bucket」
2. 填写：
   - **Bucket 名称**：全网唯一，建议格式 `xx-专业-giftbox`（如 `nkuppe-giftbox`，小写字母+数字+短横线）
   - **地域**：选最近的（华北 2（北京）/ 华东 1（杭州）等）。学生主要在国内，选近的下载快
   - **存储类型**：默认「标准存储」
   - **读写权限**：**选「公共读」**（这一步很重要，学生需要凭链接直接下载资料）
   - 其他选项默认
3. 点「确定」

**记下 Endpoint**：Bucket 创建后，进入 Bucket 详情页 → 「概览」→ 找到「Endpoint（外网访问）」字段，形如 `https://oss-cn-beijing.aliyuncs.com`。**记下来**，待会填 .env 用。

**获取 AccessKey**：
1. 阿里云控制台右上角头像 → 「AccessKey 管理」
2. **会弹警告**：「为保障账号安全，强烈建议使用 RAM 子用户」。demo 阶段可以点「继续使用 AccessKey」直接用主账号 AK；生产化时换 RAM 子账号（见下）
3. 点「创建 AccessKey」→ 验证短信/邮箱
4. 看到 **AccessKey ID**（LTAI 开头）和 **AccessKey Secret**（一长串）—— **Secret 只显示这一次，立刻复制保存到 .env，关掉窗口就再也看不到了**

**RAM 子账号（生产化推荐，demo 可跳过）**：
1. 控制台 → 「访问控制 RAM」→「用户」→「创建用户」
2. 登录名：`oss-deploy`，勾选「编程访问」（拿 AK/SK），不勾控制台访问
3. 创建后立即复制 AK/SK（同上，只显示一次）
4. 自动跳转权限页面 → 添加权限 → 搜 `AliyunOSSFullAccess` → 添加 → 确定
5. 用这个子账号的 AK/SK 填 .env（不要用主账号 AK）

### 1.3 DeepSeek API（用作 LLM，生成课程文档 + 摘要）

1. 访问 https://www.deepseek.com/
2. 注册账号（用手机号或 Google 登录）
3. 登录后顶部 → 「API Keys」→「创建 API Key」
4. 复制 `sk-` 开头的 **API Key** 保存
5. 顶部 →「充值」→ 充 10-20 元（够用很久，单次课程文档生成约 0.01 元）

### 1.4 智谱 GLM-OCR（用作 PDF OCR）

1. 访问 https://open.bigmodel.cn/
2. 注册账号（手机号）
3. 登录后 → 右上角「API Keys」→「添加新 API Key」→ 复制保存
4. 顶部 →「财务中心」→ 充值 10 元（够 OCR 几十份 PDF，单份约 0.5-2 元）

### 1.5 检查 / 装 Python 3.10+

代码用 Python 写的，需要 Python 3.10 或更新版本。**先检查你电脑有没有 Python**——通识课、数据处理课等场合你可能已经装过了。

**检查方法**：
- Windows：按 Win+R → 输入 `cmd` 回车 → 在 cmd 里输入 `python --version`
- macOS / Linux：打开终端，输入 `python3 --version`

根据结果分三种情况：

**情况 A：看到 `Python 3.10.x` 或更新**
已装好，跳过这一节，直接看 [1.6 装 LibreOffice](#16-装-libreofficeoffice-文档转-pdf-用ocr-流程必需)。

**情况 B：看到 `Python 3.9` 或更旧**
装过但版本太旧。建议升级：

Windows：
1. 访问 https://www.python.org/downloads/
2. 下载最新版
3. 双击安装，**底部勾选「Add python.exe to PATH」**
4. 控制面板 → 卸载旧版本（可选但推荐，避免冲突）
5. 重开 cmd 验证：`python --version` 应显示新版本

macOS：`brew upgrade python`（或 `brew install python@3.11`）

Linux：
- Debian/Ubuntu：`sudo apt update && sudo apt install --only-upgrade python3`
- CentOS：`sudo yum update python3`

**情况 C：蹦出来「python 不是内部或外部命令」/ 「command not found」之类的报错**（意思就是系统找不到 Python，没装）
没装 Python。装一下：

Windows：
1. 访问 https://www.python.org/downloads/
2. 点「Download Python 3.x」下载（自动识别 Windows）
3. 双击 .exe 安装，**底部勾选「Add python.exe to PATH」**（重要！不勾选 cmd 找不到 python）
4. 点「Install Now」→ 等待完成

macOS：
- 方案 A（推荐）：先装 [Homebrew](https://brew.sh/)，然后跑 `brew install python@3.11`
- 方案 B：访问 https://www.python.org/downloads/macos/ 下载 .pkg 安装

Linux：
- Debian/Ubuntu：`sudo apt update && sudo apt install python3 python3-pip`
- CentOS：`sudo yum install python3 python3-pip`

**最终验证**：终端输入 `python --version` 或 `python3 --version`，看到 3.10+ 即可。

> **默认推荐**：直接跑 setup.bat / setup.sh 即可，脚本会自动检查 Python 版本，没装或太旧都会提示你装。下面手动检查的步骤是给想自己确认版本的同学看的。

### 1.6 装 LibreOffice（Office 文档转 PDF 用，OCR 流程必需）

OCR 流程会把学生上传的 Word/Excel/PPT 文件转成 PDF 再 OCR，转换工具用免费的 LibreOffice。

**Windows**：
1. 访问 https://www.libreoffice.org/download/download-libreoffice/
2. 默认下拉选 Windows x86_64，点「下载」
3. 下载完是 .msi 文件，双击运行
4. 安装向导默认选项一路 Next →「Install」→ 等待完成
5. **验证**：按 Win+R → 输入 `cmd` 回车 → 在 cmd 里输入：
   ```
   "C:\Program Files\LibreOffice\program\soffice.exe" --version
   ```
   看到 `LibreOffice 7.x.x` 即成功

**macOS**：
- 方案 A：访问上面同一个下载页，下载 macOS 的 .dmg → 双击安装
- 方案 B（用 Homebrew）：`brew install --cask libreoffice`
- **验证**：终端输入 `/Applications/LibreOffice.app/Contents/MacOS/soffice --version`

**Linux**：
- Debian/Ubuntu：`sudo apt install libreoffice`
- CentOS：`sudo yum install libreoffice`
- **验证**：终端输入 `soffice --version`

## 二、下载项目代码 + 安装依赖

### 2.1 下载代码

打开终端（Windows: Win+R 输入 cmd；macOS/Linux: 终端），跑：

```bash
git clone https://github.com/PeberWang/PPE-CloudSmart-GiftBox.git
cd PPE-CloudSmart-GiftBox
```

如果你不用 git，也可以在 GitHub 页面点「Code」→「Download ZIP」→ 解压 → 进入文件夹。

### 2.2 一键装所有依赖 + 配置（推荐）

项目根目录有两个一键安装脚本：

- **Windows**：cd 进项目目录后，在 cmd 里直接打 `setup.bat`（PowerShell 打 `.\setup.bat`）；不会用命令行的话，文件管理器里双击 `setup.bat` 也行
- **macOS / Linux**：终端 cd 进项目目录后跑 `bash setup.sh`

脚本会自动做 4 件事：

1. **检测 Python**：没装就打开下载页提示你装（鸡生蛋限制，必须用户手动装一次）
2. **检测 LibreOffice**：没装就自动下载安装（Windows 用 winget，macOS 用 Homebrew，Linux 用 apt/yum/dnf）
3. **装 Python 依赖**：`pip install -r requirements.txt`，失败自动 fallback 到清华镜像源
4. **创建 .env 配置文件**：从 `.env.example` 复制

脚本跑完后，终端会显示「安装完成」，让你去填 .env。

**脚本失败的常见情况**：
- Windows 提示「winget 不可用」：你的 Windows 版本过旧（< 1809），手动去 [LibreOffice 官网](https://www.libreoffice.org/download/) 下载 .msi 安装
- macOS 提示「Homebrew 未装」：访问 https://brew.sh/ 装 Homebrew 后重跑
- Linux 提示「不支持的发行版」：手动装 LibreOffice

### 2.3 手动装（如果一键脚本失败）

如果你想手动控制每一步：

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows cmd:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 装 Python 依赖（此时已激活 venv，pip 装到 venv 里）
pip install -r requirements.txt

# 装失败时换清华镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 装 LibreOffice
# Windows: 访问 https://www.libreoffice.org/download/ 下载 .msi
# macOS:   brew install --cask libreoffice
# Linux:   sudo apt install libreoffice  (Debian/Ubuntu)
#          sudo yum install libreoffice  (CentOS)

# 创建 .env
# Windows cmd:
copy .env.example .env
# macOS/Linux:
cp .env.example .env
```

### 2.4 日常使用：双击 start.bat

setup 脚本把所有依赖装进了 `venv/` 目录里，**不污染你的系统 Python**。代价是：每次跑 `python deploy.py xxx` 前，要先激活 venv。

最简单的方式：**双击 `start.bat`**（Windows）或跑 `bash start.sh`（macOS/Linux）。

脚本会：
1. 自动激活虚拟环境
2. 弹出一个新的终端窗口（提示符前有 `(venv)` 标志）
3. 在这个窗口里直接跑 `python deploy.py xxx` 即可
4. 跑完后输入 `exit` 退出虚拟环境

**为什么不直接用系统 Python？**
- 虚拟环境隔离项目依赖，不和其他 Python 项目冲突
- 卸载项目时，删 `venv/` 目录即可清理干净
- 不动系统 Python 环境

## 三、配置 .env 文件

如果跑过 setup 脚本，`.env` 已经自动创建。如果没跑，手动复制：

**Windows cmd**：`copy .env.example .env`
**macOS/Linux**：`cp .env.example .env`

用记事本（或 VSCode、Notepad++）打开 `.env`，按下面填入。**等号右边 `xxxxx` 全部换成你自己的真实凭证**：

```bash
# === 飞书（1.1 节拿到）===
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_BASE_URL=https://open.feishu.cn/open-apis
FEISHU_DOC_HOST=feishu.cn

# === LLM（1.3 节拿到）===
LLM_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-v4-pro

# === OCR（1.4 节拿到）===
GLM_API_KEY=xxxxxxxxxxxxxxxx.xxxxxxxxxxxxxxxx
GLM_OCR_URL=https://open.bigmodel.cn/api/paas/v4/layout_parsing

# === 阿里云 OSS（1.2 节拿到）===
CLOUD_DRIVE_BACKEND=aliyun_oss
OSS_BUCKET=nkuppe-giftbox                          # 改成你的 bucket 名
OSS_ENDPOINT=https://oss-cn-beijing.aliyuncs.com   # 改成你的 endpoint
OSS_ACCESS_KEY_ID=LTAIxxxxxxxxxxxxxxxx
OSS_ACCESS_KEY_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OSS_PRESIGNED_TTL=86400
# 公共读直链（学生凭链接直接下载，长期有效）
OSS_PUBLIC_BASE=https://nkuppe-giftbox.oss-cn-beijing.aliyuncs.com  # 改成你的

# === 路径配置 ===
MATERIALS_BASE=./data/courses/大二上

# === 知识库 + bitable ===
WIKI_SPACE_NAME=XX 专业课程大礼包                  # 改成你的专业名
# BITABLE_APP_TOKEN 先留空，跑完 init-bitable 后填回

# === 日志 ===
LOG_LEVEL=INFO
LOG_TO_FILE=true
LOG_FILE_PATH=./logs/app.log
```

**重要提示**：
- `OSS_BUCKET` 是你 1.2 节新建的 bucket 名（小写字母）
- `OSS_ENDPOINT` 要和 bucket 的地域对应（北京 = `oss-cn-beijing`，杭州 = `oss-cn-hangzhou`）
- `OSS_PUBLIC_BASE` 格式 = `https://{OSS_BUCKET}.{去掉 https:// 的 endpoint}`
- `BITABLE_APP_TOKEN` 先不填，跑完下一步 init-bitable 后会有返回值，再回来填

保存 `.env`。

## 四、首次部署（7 条命令）

按顺序在终端跑：

```bash
# 1. 初始化 bitable（建三张管理表：课程主数据 / 资料管理 / 心得管理）
python deploy.py init-bitable
```

跑完会输出 `app_token: XXXXXXXXXXXXXX` 和 `url: https://...`。**做两件事**：
- 复制 token，回到 `.env` 文件，把 `BITABLE_APP_TOKEN=` 后面填上这个值
- **把 url 发到飞书群置顶保存**（之后管理员想编辑 bitable 都靠这个链接）

```bash
# 2. 设置 bitable 链接分享（凭链接即可访问，方便管理员 UI 操作）
python deploy.py open-bitable

# 3. 录入课程清单
```

准备一个课程清单文件（Excel/CSV/TSV 均可）。**直接填文字**，不要在 Excel 里创建下拉列表。

| 列名 | 必填 | 合法值 |
|---|---|---|
| 名称 | ✓ | 课程全称，如「概率论与数理统计」 |
| 学期 | ✓ | `大一上` / `大一下` / `大二上` / `大二下` / `大三上` / `大三下` / `大四上` / `大四下` |
| 授课老师 |  | 老师姓名 |
| 考核方式 |  | `闭卷` / `开卷` / `论文` / `其他` |
| 类型 |  | `专业必修课` / `非专业必修课`（不填默认专业必修课） |

示例（Excel 里）：

| 名称 | 学期 | 授课老师 | 考核方式 | 类型 |
|---|---|---|---|---|
| 概率论与数理统计 | 大二下 | 刘会刚 | 闭卷 | 专业必修课 |
| 微观经济学 | 大一上 |  | 闭卷 | 专业必修课 |

保存为 `data/课程清单.xlsx`（或 .csv / .tsv）。

```bash
# 批量录入
python deploy.py seed-course --from-file data/课程清单.xlsx

# 或单条录入
python deploy.py seed-course --name 概率论与数理统计 --semester 大二下 --teacher 刘会刚 --exam 闭卷
```

```bash
# 4. 建飞书知识库 + 4 个学年文档 + 内嵌课程导航表
python deploy.py wiki
# 输出里有 wiki URL，**发到飞书群置顶保存**（管理员想编辑知识库都靠这个链接）

# 5. 给自己（管理员）加知识库权限（重要！否则 UI 里没法编辑）
python deploy.py grant-wiki 你的邮箱@example.com
# 默认 admin 角色；想给只读权限加 --perm viewer
#
# 邮箱注册 → grant-wiki your-email@example.com
# 手机号注册 → grant-wiki 13800138000 --type mobile
# 或者用更简单的凭链接编辑：python deploy.py open-wiki

# 6. 生成每门课的学习指南文档（首次跑只有基本信息，资料串讲段落是占位）
python deploy.py docs

# 7. 回填导航表的「学习指南」字段（让链接可点）
python deploy.py link
```

## 4.5 保存 bitable + wiki 链接（重要）

应用是所有资源的 owner，管理员在飞书 UI「我的文档」里默认看不到这些资源。**没有链接就找不到 bitable / 知识库**，更别说编辑。

**必做**：把第 1 步的 bitable URL 和第 4 步的 wiki URL 都发到你的飞书工作群置顶保存。

理由：
- 你下次想审批学生提交的资料 → 凭 bitable 链接打开
- 你下次想编辑学年文档 → 凭 wiki 链接打开
- 团队换管理员时，新管理员凭这些链接也能接手
- 链接丢了只能重跑 init-bitable / wiki 重新建（旧数据不会丢但流程麻烦）

## 五、验证部署成功

打开飞书（用注册时的账号登录），左侧「知识库」→ 找到你刚建的 `XX 专业课程大礼包` 知识空间：

应该看到：
1. **4 个文档**（大一 / 大二 / 大三 / 大四）存在
2. 每个文档顶部有标题 + 简介 + 课程导航表
3. 导航表里每行有：课程名 / 老师 / 学期 / 类型 / 考试 / **「学习指南」可点击链接**
4. 点「学习指南」跳到该课的独立文档，里面有：
   - 授课老师 / 学期 / 考试 信息
   - 课程概述（LLM 生成，可能写「待同学补充」）
   - 推荐资料表（首次为空）
   - 资料串讲（首次是占位）
   - 贡献者表（首次为空）

如果以上都正常，**首次部署成功**。庆祝一下。

| `python deploy.py seed-course` | 录课程（单条 / --from-file 批量） |
| `python deploy.py seed-materials <dir> <course>` | 录资料（本地文件夹 → 飞书 drive + 资料表） |
| `python deploy.py archive-materials --purge-immediately` | 飞书附件 → OSS |
| `python deploy.py ocr-materials` | OCR + 摘要 |
| `python deploy.py sync` | 聚合 bitable + 门控 |
| `python deploy.py wiki` | 建/更知识库 + 学年文档 |
| `python deploy.py docs` | 生成课程文档 |
| `python deploy.py link` | 回填导航表链接 |
| `python deploy.py fix-bitable` | 修复单选字段选项 |
| `python deploy.py purge-archived` | 清理归档超期原件 |
| `python deploy.py logs` | 查看日志 |
| `python deploy.py full` | 一键全流程 |

每条命令加 `--help` 看详细参数，如 `python deploy.py seed-course --help`。
