# 02 · 学生使用指南

> 目标读者：通过表单提交资料的学长学姐。
>
> 你只需要这一篇文档就能完成资料贡献。不用装任何东西，不用懂技术。

---

## 一、你能贡献什么

| 类型 | 例子 | 备注 |
|---|---|---|
| 笔记 | 你的课堂笔记、整理的知识点 | 强烈欢迎 |
| PPT | 老师课件 | 如果老师允许分享 |
| 真题 | 往年考试题、练习题 | 强烈欢迎 |
| 教材 | 电子版教材 | 注意版权，公开出版的不建议 |
| 阅读材料 | 老师发的阅读材料、参考论文 | |
| 复习大纲 | 你整理的复习提纲 | |
| 练习题 | 习题集、答案 | |
| 其他 | 不属于以上类型的 | |

**推荐 + 必填**：每份资料都要附「推荐理由」——告诉学弟学妹**为什么推荐、怎么用最有效**。这是大礼包的灵魂。

---

## 二、提交步骤

### Step 1：拿到表单链接

问班委 / 学习委员要表单链接。打开是飞书表单页面，长这样：

```
课程资料贡献表单
─────────────────────────
贡献者：  [      ]
届别：    [下拉选]
课程：    [下拉选]
资料类型：[下拉选]
推荐理由：[多行文本]
文件附件：[拖拽文件]
```

### Step 2：填表

- **贡献者**：你的署名，建议「届别 + 姓名」（如 `22级小赵`）。这个会显示在课程文档的「贡献者」表里，让你的名字被学弟学妹看到
- **届别**：下拉选你的年级（22级 / 23级 / 24级）
- **课程**：下拉选课程名。**注意**：必须选下拉里有的；如果下拉里没有你要贡献的课，联系班委添加
- **资料类型**：下拉选（笔记 / PPT / 真题 / 教材 / 阅读材料 / 复习大纲 / 练习题 / 其他）
- **推荐理由**：核心字段。写清楚：
  - 这份资料讲什么
  - 适合什么场景（考前突击 / 日常学习 / 论文写作）
  - 怎么用最有效
  - 有什么注意事项
  - 给学弟学妹的话
- **文件附件**：拖拽文件进来。可以一次拖多份（多份资料共享同一段推荐理由完全 OK，理由混在一起写就行）

### Step 3：提交

点提交。会显示「提交成功」。然后等管理员审核 + 处理（一般 1-3 天）。

### Step 4：查收

处理完成后，你的资料会出现在课程学习指南文档里。打开知识库 → 找到对应课程 → 看到「推荐资料」表里有你的资料（带可点击的下载链接）+「资料串讲」段落里 LLM 帮你写的串讲 + 「贡献者」表里有你的署名。

## 三、写「推荐理由」的技巧

LLM 在生成课程文档时，会基于你的推荐理由 + 资料摘要生成「资料串讲」段落。理由写得越好，串讲越精彩。

### 反面例子（不推荐）

> 这份资料不错，值得看。

→ 太空泛，LLM 没法工作，串讲段落会很短

### 正面例子（推荐）

> 这份笔记覆盖了刘老师课件的所有重点章节，特别是假设检验那部分整理得很清楚。
> 适合期末冲刺复习，建议先把课件过一遍再来看这份笔记，会更顺畅。
> 注意：第四章关于回归分析的部分有点简略，建议结合教材补充。
> 给学弟学妹的话：这门课重点在应用不是推导，多做题是关键。

→ LLM 能基于这段生成连贯的串讲段落

### 多份资料共用一段理由

可以一次拖多份文件进表单，共享一段理由。比如你一次传了 4 章的练习题，理由写：

> 第一到四章的练习题合集，每章都有题目 + 详细答案。
> 建议每学完一章就做对应的练习，不要等到期末才突击。
> 答案里有解题步骤，先自己做再对答案效果最好。

→ 4 份资料会展开为 4 条 bitable 记录，共享这段理由。LLM 串讲时会基于各份资料的 OCR 摘要做差异化区分。

## 四、常见问题

### Q1：下拉里没有我的课程怎么办？

**A**：飞书表单单选字段限制，不能自由填写。联系班委 / 管理员在「课程主数据表」加这门课，加完后表单下拉会自动出现，你再重填表单。

### Q2：我上传的资料有版权问题（如电子版教材）怎么办？

**A**：教材类不建议分享公开出版物。优先分享：
- 你自己整理的笔记
- 老师发的内部材料（如果老师同意）
- 往年真题（一般老师不反对流传）
- 公开论文 / 开放获取的资料

### Q3：我可以提交心得（不附资料）吗？

**A**：可以！「心得管理表」是单独的表单（问班委要链接）。只填作者 + 课程 + 成绩 + 心得内容，不需要附件。高分心得（90+）特别欢迎。

### Q4：我提交后多久能看到？

**A**：取决于管理员处理频率。一般每周一次批量处理，1-7 天内会出现在课程文档里。

### Q5：我能改 / 删我提交的资料吗？

**A**：提交后无法直接修改。联系管理员：
- 改：管理员在 bitable UI 改对应字段
- 删：管理员在 bitable UI 删记录（OSS 上的原件会保留 7 天后自动清理）

### Q6：我提交的资料会被谁看到？

**A**：
- 管理员：bitable 里看到所有提交
- 学生：只在课程文档里看到「已通过」审核的资料
- 链接：OSS 公共读直链，任何人凭链接可下载（这是为了方便学弟学妹随时访问）

如果你不希望某份资料被广泛传播（如个人隐私 / 敏感内容），联系管理员标记为「已拒绝」或不在表单提交。

## 五、推荐理由模板（直接抄）

### 笔记类

> 这份笔记覆盖了 [课程] 老师课件的所有重点章节，特别是 [具体章节] 部分整理得很清楚。
> 适合 [期末冲刺 / 日常复习 / 论文写作]。
> 建议使用方式：[先看课件再读笔记 / 直接读笔记 / 配合做题]。
> 注意事项：[如果有简略的部分，指出来]。

### 真题类

> [年份] 的 [期中 / 期末] 真题，含详细答案。
> 老师出题风格 [偏应用 / 偏推导 / 偏记忆]，往年重复率 [高 / 中 / 低]。
> 建议考前 [1 周 / 2 周] 开始做，按时间模拟。
> 注意：[题型分布 / 重点章节]。

### PPT 类

> [老师] 老师的课件，覆盖 [学期] 全部章节。
> 老师上课风格 [讲解清晰 / 节奏快 / 互动多]，课件 [配合讲 / 主要靠课件 / 课件是辅助]。
> 建议 [上课认真听讲 / 课后过课件 / 重点章节细读]。
> 注意：[版权 / 部分章节较略]。

### 教材 / 阅读材料类

> [书名 / 论文名]，[作者 / 期刊]。
> 适合 [基础学习 / 进阶 / 拓展阅读]。
> 建议配合 [课程章节 / 老师课件] 读。
> 注意：[版权 / 阅读难度]。
