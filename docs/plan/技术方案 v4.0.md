## 一、项目重新定位

### 1.1 一句话描述

将PPE专业每学年的课程整理成一张飞书多维表格，每门课程对应一篇云文档，学弟学妹通过表格即可了解课程概况、下载学习资料。

### 1.2 核心用户旅程

```
学弟学妹打开飞书知识库
  → 看到某学年的课程多维表格（如"大二上课程表"）
    → 点击某门课程的"学习指南"链接
      → 进入该课程的专属云文档
        → 阅读课程概述、学习难点、老师偏好
        → 按分类查看和下载资料
```

**就这么简单。不需要审批流，不需要RAG问答，不需要操作日志。**

### 1.3 与v3.0的关键区别

| 维度 | v3.0 | v4.0 |
|------|------|------|
| 核心功能 | 资料收集+文档生成+审批+日志 | 课程表格→课程文档 |
| 多维表格 | 4张（资料/心得/修订/日志） | 1张（课程表） |
| 审批流程 | 宣传委员审批修订 | 暂不实现，预留接口 |
| 心得体会 | 独立管理模块 | 作为文档生成的数据源，不单独建表 |
| 数据来源 | courses.json预设127条 | 实测时按真实上传情况录入 |
| 目标 | 完整的社区协作平台 | 实用的课程资料导航 |

## 三、数据结构

### 3.1 课程多维表格（每个学年一张）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 课程名称 | 文本 | 如"中国经济概论" |
| 授课老师 | 文本 | 如"龚关老师" |
| 开课学期 | 单选 | 大一上/大一下/大二上/大二下/大三上/大三下/大四上 |
| 课程类型 | 单选 | 必修/选修 |
| 考试形式 | 单选 | 闭卷/开卷/论文 |
| 学习指南 | URL | 指向飞书知识库中的课程云文档 |
| 资料数量 | 数字 | 该课程关联的资料文件数 |
| 贡献者 | 文本 | 贡献资料的学长学姐名单 |
| 最后更新 | 日期 | 最后一次资料更新的时间 |

### 3.2 课程云文档结构（V3模板，保持不变）

1. 课程内容概述
2. 学习难点
3. 老师偏好
4. 资料分类列表
5. 资料串讲
6. 贡献者列表

### 3.3 数据来源

- **课程信息**：`config.py` 中的 `COURSES_BY_YEAR` 字典（按实测情况调整）
- **心得体会**：`data/experiences.json`（手动收集，用于LLM生成文档内容）
- **资料元信息**：`data/materials.json`（上传时记录）
- ~~courses.json~~ 已删除，不再使用预设数据

## 四、技术架构

```
deploy.py (统一入口)
  │
  ├── WikiBuilder ──── 创建知识空间 → 学年节点 → 课程节点
  │
  ├── TableService ──── 创建学年多维表格 → 填充课程记录
  │
  ├── DocGenerator ──── 读取心得+资料 → LLM生成内容 → 创建飞书云文档
  │     └── LLMService (智谱GLM-4-flash)
  │
  ├── UploadService ──── 资料上传到飞书云空间（待实测）
  │
  └── LinkService ────── 更新多维表格中的"学习指南"字段
```

### 部署模式

```bash
python deploy.py --mode full     # 完整部署（wiki + tables + docs + link）
python deploy.py --mode wiki     # 仅知识库
python deploy.py --mode tables   # 仅多维表格
python deploy.py --mode docs     # 仅生成文档
python deploy.py --mode link     # 仅关联链接
```
