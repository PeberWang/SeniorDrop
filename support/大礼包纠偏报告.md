## 一、当前代码实现了什么

### 1.1 核心功能模块

| 模块 | 文件 | 功能 |
|------|------|------|
| **资料管理** | `upload_service.py` | 扫描本地资料包，自动分类导入 |
| **心得收集** | `experience_service.py` | 心得提交、AI提取关键信息、示例生成 |
| **文档生成** | `doc_generator.py` | 基于心得+资料，调用LLM生成课程指南 |
| **飞书集成** | `feishu_service.py` | 多维表格创建、知识库创建、记录增删改 |
| **知识库构建** | `wiki_builder.py` | 创建知识空间、学年节点、课程节点 |
| **多维表格管理** | `table_service.py` | 创建4个学年表格、填充课程记录 |
| **LLM服务** | `llm_service.py` | 智谱AI调用，生成概述/难点/老师偏好等 |

### 1.2 数据流

```
本地资料包 → scan_existing_materials() → materials.json
                                        ↓
心得提交 → generate_sample_experiences() → experiences.json
                                        ↓
                            generate_course_doc() → AI生成
                                        ↓
                            输出 .md + .json 课程文档
```

### 1.3 飞书结构

```

│   ├── 世界经济概论（节点）
│   └── ...

多维表格（4个）：
├── PPE大一课程表

```

### 1.4 文档结构（V3）

## 二、与铭培理念的偏差分析

### 偏差1：课程列表硬编码，违反"非技术人员应能维护"

**铭培理念**：
> 非技术人员应能维护（增删课程、管理内容）  
> 课程列表不应写死在代码里，应从配置文件或多维表格读取

**当前代码**：
```python
# config.py 第54-84行
COURSES_BY_YEAR = {
    "大一": [
        {"name": "伦理学导论", "teacher": "李虎老师", ...},
        {"name": "宪法学", "teacher": "赵聚军老师", ...},
        # ... 所有课程硬编码
    ],
    # ...
}
```

**问题**：
- 修改课程列表需要改代码、重启程序
- 非技术人员无法自行维护
- 不同专业/学校无法复用

**严重程度**：🔴 高（违背核心设计原则）

---

### 偏差2：知识库结构偏离用户体验设计

**铭培理念**：
> 入口是飞书知识库主页  
> 导航层：4个多维表格（大一/大二/大三/大四），每个表格列出该学年各门专业课  
> 链路：知识库主页 → 选学年 → 选课程 → 读指南 → 点链接下载资料

**当前代码**：
```python
# wiki_builder.py
# 创建的是"知识库节点树"结构：
# 知识空间 → 学年节点 → 课程节点
# 
# 而4个多维表格是独立创建的，与知识库主页无关联
```

**问题**：
1. 新生进入知识库后看到的是"文件夹"式的节点树，而非直观的"表格导航"
2. 多维表格与知识库是割裂的两个系统，未形成"主页→表格→文档"的链路
3. 没有"知识库主页"的概念——知识库首页应该是4个表格的快捷入口

**严重程度**：🔴 高（影响核心用户体验）

### 偏差4：没有"一键部署"入口

**铭培理念**：
> 代码是可复用的基础设施  
> 先写代码上传GitHub，然后运行代码创建知识库

**当前代码**：
- `main.py`：扫描资料 → 生成心得 → 生成文档（本地流程）
- `pipeline.py`：创建知识库 → 创建表格 → 填充记录（飞书流程）

#### 建议2：实现资料上传到飞书云空间

**目标**：资料真正存储在飞书云空间，生成可下载链接

**方案**：
```python
# 修改 upload_service.py
async def upload_to_feishu(self, file_path, course_name):
    """上传文件到飞书云空间"""
    # 1. 调用飞书API上传文件
    result = await self.feishu.upload_file(
        file_path=file_path,
        parent_type="ccm_import_open",
        parent_token=self.folder_token
    )
    file_token = result["file_token"]
    
    # 2. 返回可访问的链接
    return f"https://nkuyouth.feishu.cn/file/{file_token}"
```

**改动点**：
- `upload_service.py` 添加飞书上传逻辑
- `doc_generator.py` 在资料列表中嵌入下载链接
- 初始化时创建飞书云空间文件夹

#### 建议3：实现文档上传到飞书并关联知识库

**目标**：AI生成的文档自动上传到飞书云文档，并关联到知识库节点

**方案**：
```python
# 修改 doc_generator.py
async def upload_doc_to_wiki(self, course_name, md_content, year):
    """上传文档到飞书知识库"""
    # 1. 创建飞书云文档
    doc_result = await self.feishu.create_doc(course_name, md_content)
    doc_token = doc_result["document"]["document_id"]
    
    # 2. 获取知识库节点token
    node_token = self.wiki_builder.get_course_node_token(year, course_name)
    
    # 3. 将文档关联到知识库节点
    await self.feishu.link_doc_to_wiki_node(
        space_id=self.wiki_builder.space_id,
        node_token=node_token,
        obj_token=doc_token
    )
    
    # 4. 更新多维表格中的"学习指南"字段
    await self.table_service.update_guide_link(
        year=year,
        course_name=course_name,
        guide_url=f"https://nkuyouth.feishu.cn/wiki/{node_token}"
    )
    
    return doc_token
```

### 3.2 功能补充（优先级：🔴 高）

#### 建议5：创建统一部署入口

**新增文件**：`deploy.py`

```python
"""
一键部署脚本
"""
async def deploy():
    """完整部署流程"""
    print("🚀 开始部署PPE云端智能大礼包...")
    
    # Step 1: 初始化飞书服务
    feishu = FeishuService()
    
    # Step 2: 从飞书读取课程列表（或使用默认配置）
    courses = await load_courses()
    
    # Step 3: 创建知识库结构
    wiki = WikiBuilder(feishu)
    await wiki.build_all()
    
    # Step 4: 创建多维表格
    tables = TableService(feishu)
    await tables.create_all_tables()
    
    # Step 5: 上传资料到飞书云空间
    uploader = UploadService(feishu)
    await uploader.upload_all_materials()
    
    # Step 6: 生成并上传课程文档
    doc_gen = DocGenerator(feishu)
    await doc_gen.generate_and_upload_all(courses, wiki, tables)
    
    # Step 7: 关联表格与文档
    await tables.link_all_guides(wiki)
    
    print("✅ 部署完成！")
    print(f"📚 知识库地址: https://nkuyouth.feishu.cn/wiki/{wiki.space_id}")
```

**命令**：
```bash
python deploy.py                    # 完整部署
python deploy.py --skip-upload      # 跳过资料上传（已有资料）
python deploy.py --only-docs        # 只重新生成文档
```

## 五、总结

### 当前代码的问题

1. **定位偏差**：代码更像是"技术原型Demo"，而非"可部署的生产系统"
2. **核心链路断裂**：资料→文档→表格→知识库没有打通
3. **运维门槛高**：非技术人员无法维护课程列表

### 纠偏方向

1. **打通飞书全链路**：资料上传→文档生成→知识库关联→表格链接
2. **外置配置**：课程列表、资料路径等都从飞书或配置文件读取
3. **一键部署**：提供 `deploy.py` 统一入口

### MVP目标达成路径

```
当前状态:
  代码能生成文档（本地）
  代码能创建飞书结构（空壳）
  ↓
P0完成后:
  代码能生成文档并上传飞书
  代码能上传资料到飞书云空间
  新生能看到文档、点击下载资料
  ↓
MVP达成:
  2026年9月新生入学时
  打开飞书知识库 → 选课程 → 看指南 → 下载资料
```
