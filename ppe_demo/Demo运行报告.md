## 三、技术实现

### 3.1 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 语言 | Python 3.14 | 主要开发语言 |
| AI服务 | 智谱AI GLM-4-Flash | 文档生成、信息提取 |
| 异步处理 | asyncio | 异步API调用 |
| HTTP客户端 | httpx / zhipuai SDK | API请求 |

### 3.2 代码结构

```
ppe_demo/
├── config.py                 # 配置管理
├── models.py                 # 数据模型（Material, Experience, CourseDocument）
├── main.py                   # 主流程
├── requirements.txt          # 依赖包
├── data/                     # 数据目录
│   ├── courses.json         # 课程列表
│   ├── materials.json       # 资料元数据（127条）
│   ├── experiences.json     # 心得体会（4条）
│   └── report.json          # 汇总报告
├── services/                 # 服务模块
│   ├── upload_service.py    # 上传服务（扫描、归档）
│   ├── experience_service.py # 心得服务（提交、提取）
│   ├── doc_generator.py     # 文档生成（6部分）
│   └── llm_service.py       # LLM服务（智谱AI）
├── output/                   # 输出目录
│   └── course_docs/         # 课程文档（18个文件）
└── README.md                 # 使用说明
```

### 3.3 核心流程

```python
# 步骤1: 扫描现有资料包
upload_service.scan_existing_materials()
# → 扫描9门课程、127个文件
# → 自动识别资料类型
# → 标准化命名：{类型}_{贡献者}_{原始文件名}

# 步骤2: 生成示例心得体会
experience_service.generate_sample_experiences(courses)
# → 生成4条心得
# → AI提取关键信息（概述、难点、老师偏好）
# → 自动审核通过

# 步骤3: 生成课程文档
doc_generator.generate_all_course_docs(courses)
# → 为每门课程生成6部分文档
# → 基于心得体会和资料列表
# → 输出Markdown和JSON格式
```

## 五、发现的问题与解决

### 5.1 PDF生成失败
- **问题**：`page.pdf_output()` API不存在
- **解决**：改为 `page.pdf()`
- **状态**：✅ 已修复

### 5.2 智谱AI 401错误
- **问题**：旧API Key失效
- **解决**：更新为新的API Key（已移至 .env 环境变量管理）
- **状态**：✅ 已修复

### 5.3 首次运行部分文档为空
- **问题**：数据写入与加载时序问题
- **解决**：重新生成文档
- **状态**：✅ 已解决

### 5.4 Windows控制台编码问题
- **问题**：GBK编码无法显示emoji和中文
- **解决**：使用 `io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')`
- **状态**：✅ 已修复
