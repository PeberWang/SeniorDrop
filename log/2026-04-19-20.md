# 2026-04-19-20 B方案三合一改造

## 改动文件

### 1. `ppe_demo/config.py` — B方案2：路径配置灵活性

- 新增 `_resolve_path()` 函数，统一处理路径解析
- 解析优先级：环境变量 → .env文件 → 绝对路径默认值 → 相对路径默认值
- `MATERIALS_BASE` 支持相对路径 `materials/`（相对于项目根目录）
- `COURSE_REFORM_NOTES_DIR` 支持相对路径 `course_reform_notes/`
- 新增 `PROJECT_ROOT` 常量指向 `PPE-CloudSmart-GiftBox/`
- 新增 `PROTECTED_FIELDS` 集合（`{"贡献者", "最后更新"}`），供增量更新使用

### 2. `ppe_demo/services/table_service.py` — B方案3：增量更新策略

- `populate_all_tables()` 新增 `incremental` 参数（默认 True）
- 新增 `_populate_incremental()` 方法：
  - 先读取已有记录，构建课程名称→record映射
  - 只新增 config 中有但表格中没有的课程
  - 只更新有字段变化的已有课程
  - 跳过 `PROTECTED_FIELDS` 中的字段（贡献者、最后更新），保护用户手动编辑
- 新增 `_populate_full()` 方法：全量覆盖（原有逻辑）
- 新增 `load_config()` 方法：从 bitable_config.json 加载已有配置
- `create_all_tables()` 改为：已有表格跳过创建，只补全缺失学年
- 新增 `_field_to_str()` 静态方法：统一飞书字段值到字符串的转换
- 读取记录失败时自动回退到全量模式

### 3. `deploy.py` — 新增 sync 模式

- 新增 `--mode sync`：增量同步课程记录，不覆盖用户修改
- `deploy_tables()` 新增 `incremental` 参数，默认增量
- 新增 `deploy_sync()` 函数
- 更新帮助文档，说明环境变量用法

### 4. `.env.example` — 更新路径配置说明

- 路径变量改为注释示例，说明支持相对路径
